GitHub Pages
------------

O famosissimo GitHub Pages foi desenvolvido pelo próprio GitHub, com o intuito de fazer com que cada projeto ou usuário tenha sua página para poder 

se apresentar e explicar algumas informações do código do projeto. Mas se usarmos da maneira "certa", poderemos usar o sistema para podermos ter um 

blog interamente hosteado de graça.

Com o GitHub pages, você pode configurar o seu apontamento de domínio, ou seja, usar o seu próprio domínio para colocar seu site no ar. Pode usar o 

Pelican para gerar os seus conteúdos estáticos, fazendo com que a pasta com os HTMLs tenha um Commit automático e auto-publicando seu novo conteúdo 

a cada vez que for gerado.

CloudFlare
----------

Um dos melhores sistemas de CDN e de hospedagem de zona DNS do MUNDO, o CloudFlare permite com que você configure uma zona DNS de categoria Premium, 

sem pagar um tostão furado. Exatamente de graça, sem precisar chorar com o dono do sistema e nada. Caso você queira algumas mordomias, como dominio 

personalizado e etc em sua zona, você pagará o valor que inicia em 20 dólares.

Com esta zona criada e apontada corretamente aos servidores do GitHub, você poderá usar o seu domínio sem problemas.

Domínio
-------

Como muitos dizem, não existe nenhuma forma de fugir do valor de 30 reais do domínio .com.br, ou algum outro valor cobrado nos registrar que existem 

e dominam o mercado. Mas graças a uma pequena pesquisa, eu descobri que a extensão(TLD) .TK é gratuito para qualquer tipo de uso. Para registrar um 

domínio .TK, você terá de acessar este link: http://www.dot.tk/pt/index.html
e ver se o domínio de sua escolha está disponível para registro.

Caso você não se importe de gastar um valor anualmente, recomendo ver o seu domínio no Domai.NR, ele é um site que permite que você também escolha 

um link curto para você hospedar o seu blog além de habilitar a escolha de outro TLD caso não esteja disponível.

Hosting de Imagens
------------------

Esta é uma das coisas que pesa muito no quesito de manter um site, e que alguns serviços de Hosting cobram se você baixar muito. Isso pode se 

agravar se você hosteia imagens em seu servidor, aumentando o seu uso de banda e permitindo com que você gaste o seu dinheiro com isso.

Para cortar o seu custo com este tipo de Hosting, eu recomendo o IMGUR, um site que você só necessita de uma conta e ele te dará um espaço ilimitado 

para hosting de imagens, deixando a quantidade apenas a sua escolha, e não ao do servidor.


Hosting de Áudio
----------------

Para esta proesa, existe também uma solução para hospedar aúdio de graça e sem se preocupar se o servidor está online ou não, pois é um dos 

servidores mais estáveis da Web, se não o mais estável. O Archive.org é um sistema que permite a qualquer usuário hospedar hospedar aúdios, vídeos e 

textos para permitir que ele se mantenha na web por tempo indeterminado.

O único problema dele é que todo o conteúdo dele tem que ser produzido por você, não permitindo que você insira conteúdos de terceiros sem a 

permissão prévia.


Conclusão
---------

É realmente possível manter um site online sem ter custos com nada e sem se preocupar muito, além de alimentar o conteúdo dele. Retirando esta parte, vemos que todos os sites podem também ter as melhorias para poder atender o mercado e fornecer aos leitores a melhor experiência se quiser/puder gastar um pouco.
